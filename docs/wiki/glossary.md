# Glossary

[← wiki index](README.md)

> Every domain term used in this codebase, defined. Alphabetical.

---

**A2UI** — Agent-to-UI v0.8 protocol (per [ADR-0005](adrs.md#adr-0005-genui-protocol)). Wire format for agents emitting UI patches; the `a2ui.delta` event wraps an A2UI message in the canonical event envelope. Schema: `packages/protocol/schemas/events/a2ui-delta.schema.json`. See [generated-ui surface](surfaces/generated-ui.md).

**AccessCode** — Single-use code for private user onboarding (per [ADR-0010](adrs.md#adr-0010-tenant-access-control--access-codes)). **Not implemented in code.** 🗑️ skipped for hackathon.

**Adapter (resident-runner)** — Pluggable execution engine inside a `ResidentRunner`. Modes: `smoke` (canned text) and `hermes-cli` (spawns the `hermes` binary). Configured by `AGENTS_RESIDENT_ADAPTER`.

**Admin console** — Web UI at `/admin` route showing runs, runner fleet, lineage, and agent workshop. Gated by `ADMIN_EMAILS` env on Control API (currently hardcoded to `seb4594@gmail.com`). See [admin-console reference](reference/admin-console.md).

**AgentInstance** — A logical agent running inside a ResidentRunner. Identified by `agentId` within `(runnerId, userId)` boundary. Tracked in `AgentInstancesTable` (table provisioned, **not currently written**).

**AgentProfile / AgentProfileVersion** — Versioned specification for a specialist agent: role, mission, behavior, tool policy, MCP policy, eval pack, scorecard, approval block. Schema in `packages/agent-profile/src/types.ts`. Stored in `AgentProfilesTable`. Bundle artifacts in S3.

**AgentProfileLifecycle** — State machine for an AgentProfileVersion: `draft → review → approved → promoted → retired`. Validation gates each transition.

**Agent runtime** — The ECS Fargate worker that executes runs. Two modes: stateless (live) and resident-runner (built but unwired). See [agent-runtime](services/agent-runtime.md).

**Agent workshop** — UI panel for drafting and approving AgentProfileVersions. Two implementations: `services/agent-creator/` (CLI) and `apps/web/lib/agent-workshop.ts` (web). See [agent-creator reference](reference/agent-creator.md).

**Amplify** — AWS Amplify Gen 2. Owns the Cognito user pool (`us-east-1_1UeU1hTME`) and hosts the web frontend at `https://main.dkqxgsrxe1fih.amplifyapp.com/`. CDK imports the user pool. See [secondary-infra](infrastructure/secondary-infra.md).

**API Gateway** — AWS managed HTTP/WebSocket API. Hosts Control API (HttpApi) and Realtime API (WebSocketApi). JWT/Lambda authorizer in front of every route.

**Approval / tool.approval** — Human-in-the-loop gate before a risky tool executes. Schema: `tool-approval.schema.json`. Discriminated request/decision payloads. Currently emitted only by local harness, not the live worker.

**Artifact** — File or report produced by a run. Stored in S3 + ArtifactsTable + emitted as `artifact.created` event. Kinds: `document, website, dataset, report, diff, miro-board, log, trace, other`. See [artifacts surface](surfaces/artifacts.md).

**A2UI** — see "A2UI".

**Bundle (profile bundle)** — Multi-file representation of an AgentProfileVersion (profile.json, SOUL.md, config.fragment.yaml, skills/, policy/, evals/, scorecards/, manifest.json with sha256 hashes). Generated by agent-creator's `--bundle-dir` flag. The Control API persists a single-file `profile.json` artifact in S3, not the full bundle.

**Canonical event envelope** — The wire format every cross-service event uses. Defined in `packages/protocol/schemas/event-envelope.schema.json`. Required fields: `id, type, seq, createdAt, orgId, userId, workspaceId, runId, source, payload`. See [protocol reference](reference/protocol-package.md).

**CandidateTool** — Discovery-time tool descriptor used by the workshop. `{id, name, category, risk, description, source, catalogHash}`. Lifecycle: candidate → bound into AgentToolPolicy entry.

**CDK** — AWS Cloud Development Kit. Source for all 10 infrastructure stacks. Entry: `infra/cdk/src/bin/agents-cloud-cdk.ts`. See [stacks](infrastructure/stacks.md).

**Cloudflare Realtime** — Worker + Durable Objects scaffold at `infra/cloudflare/realtime/`. **Deferred for hackathon.** AWS-native realtime is the live primary path.

**Cognito** — AWS Cognito User Pool, the identity provider. Owned by Amplify (not CDK). Pool ID `us-east-1_1UeU1hTME` hardcoded in CDK config defaults. JWT `sub` claim is the canonical `userId` everywhere.

**Cognito groups** — Per [ADR-0010](adrs.md#adr-0010-tenant-access-control--access-codes): `agents-cloud-user`, `-admin`, `-suspended`, `-internal`. **Not implemented.** 🗑️ skip.

**Command center** — Web home page (`apps/web/components/command-center.tsx`). Where users submit objectives and watch live run timelines.

**Control API** — HTTP API for all run/work-item/runner-state CRUD. Live URL: `https://ajmonuqk61.execute-api.us-east-1.amazonaws.com`. See [control-api](services/control-api.md).

**DataSourceRef** — External-data reference (artifact pointer, inline, third-party). Table provisioned, schema undefined, routes 501. See [data-sources surface](surfaces/data-sources.md).

**Durable Object (DO)** — Cloudflare Workers primitive: a single-threaded actor with persistent storage. `UserHubDO`, `WorkspaceHubDO`, `SessionHubDO` in the deferred Cloudflare scaffold.

**ECS Fargate** — AWS managed container compute. Used for both `agent-runtime` (stateless worker, live) and `resident-runner` (built but unwired) Fargate task definitions.

**EFS** — AWS Elastic File System. Optional POSIX-shared filesystem for hot workspaces. **Deferred indefinitely.** S3 is the durable store.

**Event** — A `CanonicalEventEnvelope` row in `EventsTable`. Currently produced types: `run.status`, `artifact.created`, `tool.approval` (harness only). Reserved: `a2ui.delta`. See [events catalog](reference/events-catalog.md).

**Event-relay** — Service to bridge AWS DDB streams → Cloudflare. README only. **Not built.** 🗑️ skip.

**Eval pack** — A set of `AgentEvalScenario`s tied to an AgentProfileVersion. Used to gate promotion. Validator requires non-empty scenarios. **No automated runner exists.**

**Fargate** — see "ECS Fargate".

**GenUI** — "Generated UI" — the broad concept of agents shipping UI patches to clients. Implemented via A2UI v0.8 messages. See [generated-ui surface](surfaces/generated-ui.md).

**Hermes** — A CLI tool the worker can shell out to (`hermes chat -q <prompt>`). The image **does not currently bake in** the Hermes binary, so `HERMES_RUNNER_MODE=cli` would fail at spawn. Default mode is `smoke` (no real model). See [agent-runtime](services/agent-runtime.md).

**HostNode** — A registered host capable of hosting ResidentRunners. Heartbeats via `POST /runner-hosts/{hostId}/heartbeat`. Stored in `HostNodesTable`.

**Idempotency scope** — Composite key `${userId}#${workspaceId}#${idempotencyKey}` used to deduplicate `POST /runs` and `POST /work-items`. See [idempotency reference](reference/idempotency.md).

**Inbox / wake** — In ResidentRunner: a wake() invocation triggers an agent to act on an objective. Currently HTTP-only on-demand; no inbox/timer scheduler exists.

**JWT** — JSON Web Token issued by Cognito. ID token attached to all client requests. Bearer header for HTTP, `?token=` query string for WebSocket.

**Lifecycle (runner)** — `idle → starting → running → succeeded / failed / drained`. Tracked in `UserRunnersTable.status`.

**Lifecycle (profile)** — see "AgentProfileLifecycle".

**Lineage timeline** — Admin UI panel that orders run events with human summaries and pipeline-position diagnostics. See [admin-console reference](reference/admin-console.md).

**Local harness** — Deterministic scripted multi-agent simulation in `services/agent-runtime/src/local-harness.ts`. The only end-to-end multi-agent flow in the codebase. See [local-harness reference](reference/local-harness.md).

**MCP** — Model Context Protocol. AgentMcpPolicy pins allowed MCP servers (with `pinnedDefinitionHash`) and gates `allowDynamicServers`.

**Miro bridge** — Service to integrate Miro boards. README only. **Not built.** 🗑️ skip.

**Objective** — Free-text user input passed as `OBJECTIVE` env var to the worker. The thing the agent is supposed to do.

**OrgId** — Organization scope identifier. Required field in canonical event envelopes; **currently dead** — `create-run.ts` doesn't populate it; protocol builder defaults to `org:${userId}`.

**Preview hosting** — Wildcard subdomain hosting for agent-generated static sites (per [ADR-0007](adrs.md#adr-0007-preview-hosting)). PreviewIngressStack uses upstream nginx as placeholder. **Disabled for hackathon.** 🗑️ skip.

**Protocol package** — `@agents-cloud/protocol`. Owns JSON Schemas + TS builders for canonical events. See [protocol reference](reference/protocol-package.md).

**Realtime API** — AWS WebSocket API + DDB-stream relay. Live wss:// URL, JWT-authorized. See [realtime-api](services/realtime-api.md).

**Resident runner** — Per-user ECS container hosting many logical agents. The hackathon's target architecture. **Image + TaskDef built; no scheduler/dispatcher exists.** See [agent-runtime](services/agent-runtime.md).

**Run** — Execution unit. Created by `POST /runs`. Has runId, taskId(s), status, events, artifacts. Stored in `RunsTable`. See [runs-and-tasks surface](surfaces/runs-and-tasks.md).

**Run ledger** — Web client view aggregating events into a timeline. `apps/web/lib/run-ledger.ts`.

**Runner** — Generic term for an ECS task that runs an agent. Two kinds: `agent-runtime` (stateless, per-run) and `resident-runner` (per-user, multi-agent).

**Runner snapshot** — S3-persisted state of a ResidentRunner. Table `RunnerSnapshotsTable` provisioned, **never written.**

**Scorecard** — `AgentProfileScorecard` — quality metrics on a profile draft. `readyForUserReview`, `readyForPromotion`, blocker count, etc. Validation produces it.

**Seq** — Monotonic positive integer identifying an event's position in a run's event log. Hardcoded `2,3,4` in worker today (fragile under retry).

**Smoke worker / smoke mode** — Default `HERMES_RUNNER_MODE=smoke`. Returns canned text without calling a model. Live in production today. ⚠️ Replace with real model call for hackathon.

**Step Functions** — AWS state machine service. `simple-run` is the only state machine — single-state, calls `ecs:runTask.sync` against `agent-runtime` task family.

**Surface** — A persistent A2UI/GenUI-driven UI presented to users. Stored in `SurfacesTable`. Routes 501. See [generated-ui surface](surfaces/generated-ui.md).

**Task (control plane)** — Sub-unit of a Run. Has workerClass, status, retries. Stored in `TasksTable`. No client UI yet.

**Topic (realtime)** — Subscription key in `RealtimeConnectionsTable`. Format: `pk=TOPIC#run:{workspaceId}:{runId}`. Cross-user fanout filtered by `event.userId`.

**Tool catalog** — Set of `CandidateTool` definitions discoverable by an AgentProfile. Risk levels: `low / medium / high`. High/medium tools must be `approvalRequiredTools`.

**UserRunner** — Row in `UserRunnersTable` describing a per-user resident runner. `(userId, runnerId)` keyed. Has `desiredState` and `status`. **Provisioned, no actuator.**

**Workspace** — Logical container for a user's work. Currently every user shares `workspaceId: "workspace-web"` (hardcoded in web). Membership/ACL model deferred.

**WorkItem** — Objective layer above runs. Multiple Runs can attach. Stored in `WorkItemsTable`. API real, clients fixture-only. See [work-items surface](surfaces/work-items.md).
